11. Java虚拟机

Sun classic vm

已经被淘汰，但是是第一款商用虚拟机。
只能使用纯解释器方式执行java代码。

Exact VM

Exact MemoryManagement 准确式内存管理。
编译器和解释器混合工作模式。
性能比classic VM有提高。但也已经被淘汰。

HotSpot VM！！！

目前使用最多的java 虚拟机产品。
性能高。

KVM

kilobyte 简单，轻量，高度可移植。在手机平台运行，但是运行速度很慢。嵌入式虚拟机。

JRockit ！！

BEA公司研发。JRockit是世界上最快的Java虚拟机。专注服务器端应用。
优势：垃圾收集器， MissionControl服务套件。该套件用来诊断内存泄漏并指出原因。

J9

IBM研发。
IBM Technology for Java virtual Machine IT4j.
类似于HotSpot。

Dalvik

google研发。不是java虚拟机。适用于安卓开发。

Microsoft JVM

被oracle告上法庭。

Azul VM 和Liquid VM

这两个是高性能的java虚拟机。
Liquid Vm 本身不需要操作系统的支持。

2.2 Main()函数

​ Main()函数修改如下。Main函数输入的参数包括String类型的method和Object类型的其他参数。

​ Neo智能合约已经实现的触发器有两种，Verification和Application。Verification触发器的目的是将合约作为验证函数进行调用，并且返回有效的bool值。Application触发器的目的是将合约作为应用函数进行调用，应用函数可以接受多个参数吗，对区块链的状态进行更改。if(Runtime.Trigger == TriggerType.Application)首先判断当前合约是否是Application应用函数。如果是则根据参数method执行不同的方法，包括deploy(), name(), symbol(), decimals(), totalSupply(), balanceOf(), transfer()。

###2.3 Name()方法和Symbol()方法

​ Name()方法返回资产的名称，Symbol()方法返回资的符号。本例分别为“ZTOKEN”和“Z”。

2.4 Deploy()方法

​ Deploy()函数只能由superAdmin执行，如果由其他用户调用，则返回false。当superAdmin调用Deploy（）函数的时候，定义一个StorageMap用于存储资产。Map的key值是账户，value值是账户对应的余额。代币总金额为3000000， 精度为8位小数。Deploy()函数把“total supply”和totalsupply作为一对键值对加入Map，把”superAdmin”和totalDeploy作为一对键值对加入Map，表示把所有资产归到superAdmin名下。

2.5 TotalSupply()方法和Decimals()方法

​ TotalSupply方法定义资产总量为3000000个Z，Decimals()函数返回资产的精度为8。

2.6 BalanceOf()方法

​ BalanceOf 函数从asset 的Map中取出账户对应的余额。

2.7 Transfer()方法

​ Transfer（）函数的参数是转出地址from，转入地址to和转账金额value。如果value的值为负，则返回false。 Transfer函数必须由转出地址from调用执行。CheckWintess（from）检测调用者是否是from，如若不是，则返回false。如果调用者是from账户，则需要检测账户from的余额是否大于等于转账金额value。 BigInteger fromValue = asset.Get(from).AsBigInteger()从asset Map中获取账户from的余额，然后进行判断。

​ 当以上条件都满足的时候，则让账户from的余额减少value，账户to的余额增加value，重新把这两个键值对put到Map中。然后执行事件Transfered（from， to， value）
